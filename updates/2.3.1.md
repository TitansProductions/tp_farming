# What has been updated?

## Changelog - (Added, Fixed, Removed):

1. Fixed an issue related to a task that was running 0.0 ms which should not, that happened because of several changes that i had made in previous updates.
2. Removed `Config.AddUnfinishedPlantMarker` option in the configuration file which was not working properly, now every plant will be having its own marker configuration option to display the marker properly. This has been changed because the plants are not having the same height and size and the marker was not visible in most cases.
3. Added plant rotation feature because there were some feedbacks related to it, many people wanted a rotation system for the plants placement.

## Notes

1. Add in every plant configuration after `Animation` option the new option:

```lua
        -- Add a circular marker on plants for being more visible?
        AddMarker = {  -- version '2.3.1'
        
            -- Set to true to enable this system in general.
            Enabled = true,

            -- Set to true if you want to display the marker only when the plant is not "planted" on all of its stages, 
            -- otherwise set to true if you want always to be visible.
            DuringPlantingOnly = true,

            RGBA = {r = 255, g = 255, b = 255, a = 50}, 
            DisplayDistance = 5.0,

            -- @param type : "+", "-" (this is for increasing or decreasing the marker display "+" UP, "-" = DOWN)
            HeightAdjustment = { type = '-', height = 0.7 },
        },
```

## How to update?

- You can easily update the modified files by redownloading from the **Cfx.re Portal** and copy-paste on the mentioned directory file paths.
- If new options have been implemented to the configuration file or edited, **DO NOT** copy-paste the new configuration file, otherwise you will get the latest configuration but you will lose all the changes you have made. The proper way updating a configuration file is by finding the new configuration options or the modified ones and do it **manually**.

*Cfx.re Portal **DOES NOT** force-update your configuration file or any files, if it did, you would always lose all the configuration changes you made or would totally break the script, all files are updated or edited manually.*
